# StoryDeck Demo 演示脚本

## 1. 演示目标

用 5-8 分钟展示 StoryDeck 如何把传统 PPT 编辑器的“逐页制作”工作流，改造成 AI Native 的“叙事结构驱动”工作流。

演示时重点不是展示所有功能，而是证明两个判断：

1. AI 生成 PPT 后，用户最需要调整的是叙事结构和表达意图，而不是马上进入像素级编辑。
2. 当工具保存了结构和意图，AI 就可以持续、局部、可控地协作。

## 2. 本地运行

```bash
npm install
npm run dev
```

默认访问：

```text
http://127.0.0.1:5173/
```

如果要展示真实 PPT 预览，在第二个终端运行：

```bash
npm run preview-server
```

## 3. 推荐演示路径

### Step 1：打开工作台，先展示 Narrative Map

讲述点：

- 传统 PPT 编辑器左侧是 slide 缩略图。
- StoryDeck 左侧是 Narrative Map。
- 这里的最小单位不是“第几页”，而是“这段演讲承担什么表达任务”。

可展示内容：

- 左侧节点角色：开场、共鸣、冲突、论证、转折、行动、收束。
- 每个节点有表达意图和时长。
- 中间预览页和左侧节点联动。

### Step 2：打开 Settings，展示全局生成入口

讲述点：

- 新建叙事地图是全局操作，不放在每一页 Intent Panel 里。
- 用户从主题、听众、目标、时长开始，而不是从空白页开始。
- AI 先生成结构，再生成模板。

可展示内容：

- 主题、听众、目标、时长输入。
- OpenAI-compatible API 设置。
- 当前模板锁定状态和色板。
- 版本管理。

### Step 3：选择一个节点，编辑表达意图

建议选择“问题引入”节点。

把表达目标改成类似：

```text
用一个具体失败交易故事开场，让听众先感受到信任和履约成本。
```

讲述点：

- 用户不是直接命令 AI “美化这一页”。
- 用户告诉系统这一页的表达任务。
- 这使 AI 重写有明确上下文。

### Step 4：点击 AI 重写，展示建议稿对比

讲述点：

- AI 不直接覆盖原稿，而是生成建议稿。
- 用户仍然保留判断权，可以应用或放弃。
- 这比传统“一键生成整份 PPT”更适合真实工作，因为真实用户会反复局部调整。

可展示内容：

- 原稿和 AI 建议稿对比。
- 点击应用后，当前页内容更新。
- 叙事结构和模板保持不变。

### Step 5：切换当前页布局

在预览工具栏选择：

- 场景陈述
- 三点论证
- 流程路径
- 行动收束

讲述点：

- 布局是表达结构的一部分，不只是视觉模板。
- 页面内容、布局注册表、PPTX 导出共用同一套结构定义。
- 这解决了 Web 预览和导出不一致的问题。

### Step 6：点击撤销

讲述点：

- AI Native 工作流需要允许用户试错。
- 用户敢让 AI 参与修改，是因为可以撤销和保存版本。
- 撤销针对 Deck 级编辑，而不是只撤销一个文本框。

### Step 7：展示 LibreOffice 预览服务状态

如果预览服务未启动：

- 点击“预览服务状态”。
- 展示启动命令 `npm run preview-server`。

如果预览服务已启动：

- 展示当前页来自 LibreOffice 渲染结果。

讲述点：

- PPT 是交付型文件，用户最终要看 PowerPoint 中的结果。
- 所以 StoryDeck 不只做 CSS 预览，而是尝试接近真实 PPT 渲染。

### Step 8：导出 PPTX

讲述点：

- 最终交付仍然是 PPTX。
- StoryDeck 不是替代所有 PowerPoint 能力，而是重构 AI 生成到成稿之间最痛的编辑阶段。

## 4. 与传统编辑器的差异总结

演示结束时可以用这段话收束：

```text
传统 PPT 编辑器默认用户从空白页开始逐页制作，所以它的核心交互是页面列表、画布和对象属性。

StoryDeck 默认用户从 AI 初稿开始，需要判断和调整表达结构，所以它的核心交互是 Narrative Map、Intent Panel 和 AI 局部重写。

用户编辑的不是孤立页面，而是整套 Deck 的叙事结构；AI 响应的也不是单个 prompt，而是带有目标、角色、意图、布局和模板约束的上下文。
```

## 5. 演示风险和备用方案

### AI 接口不可用

如果现场网络或 API key 不稳定，可以用已有示例 Deck 演示：

- Narrative Map
- 意图编辑
- 布局切换
- 撤销
- 版本管理
- LibreOffice 预览状态
- PPTX 导出

然后说明 AI 生成和重写流程已在代码中实现，并有测试覆盖。

### LibreOffice 服务不可用

如果没有安装 LibreOffice 或服务未启动，可以展示：

- 预览状态提示
- 启动命令
- CSS 回退预览

并说明产品设计上已经为真实 PPT 渲染预留本地服务。

### 导出文件现场不打开

如果现场不方便打开 PowerPoint，可以只展示导出按钮和项目 README 中的导出说明。核心评分点仍然是交互范式，而不是完整办公套件能力。
